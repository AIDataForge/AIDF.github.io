1. 概述

AI NAS 提供完整的 RESTful API 接口，基于 FastAPI 框架实现。所有 API 接口都支持异步处理，返回 JSON 格式数据。

1.1 基础信息

  • Base URL：http://localhost:8000（开发环境）
  • API前缀：/api/v1
  • Content-Type：application/json
  • 响应格式：统一使用 JSON 格式

1.3 状态码

  • 200：成功
  • 400：请求参数错误
  • 401：未授权
  • 403：禁止访问
  • 404：资源不存在
  • 500：服务器内部错误

2. API分类

2.1 认证API

提供用户注册、登录、登出、权限验证等功能。

主要接口：

  • POST /api/v1/auth/register - 用户注册
  • POST /api/v1/auth/login - 用户登录
  • POST /api/v1/auth/logout - 用户登出
  • GET /api/v1/auth/me - 获取当前用户信息
  • GET /api/v1/auth/permissions - 获取用户权限
  • POST /api/v1/auth/change-password - 修改密码

📖 查看详细API文档 →

2.2 文件管理API

提供文件上传、下载、列表查询、详情获取等功能。

主要接口：

  • POST /api/v1/files/upload - 上传文件
  • GET /api/v1/files/ - 获取文件列表
  • GET /api/v1/files/{file_id} - 获取文件详情
  • GET /api/v1/files/{file_id}/download - 下载文件
  • DELETE /api/v1/files/{file_id} - 删除文件

📖 查看详细API文档 →

2.3 文档处理API

提供文档处理状态查询、处理队列管理等功能。

主要接口：

  • GET /api/v1/processing/status - 获取处理状态
  • GET /api/v1/processing/queue - 获取处理队列
  • POST /api/v1/processing/process/{file_id} - 触发文档处理

📖 查看详细API文档 →

2.4 智能对话API

提供文档对话、智能问答等功能。

主要接口：

  • POST /api/v1/chat2doc/chat - 文档对话
  • POST /api/v1/chat2doc/stream - 流式对话
  • GET /api/v1/chat2doc/history - 获取对话历史

📖 查看详细API文档 →

2.5 项目管理API

提供项目管理、成员管理、记录管理等功能。

主要接口：

  • GET /api/v1/projects/ - 获取项目列表
  • POST /api/v1/projects/ - 创建项目
  • GET /api/v1/projects/{project_id} - 获取项目详情
  • POST /api/v1/projects/{project_id}/members - 添加项目成员

📖 查看详细API文档 →

2.6 系统管理API

提供用户管理、权限管理、系统设置等功能。

主要接口：

  • GET /api/v1/user-management/users/ - 获取用户列表
  • POST /api/v1/user-management/users/ - 创建用户
  • GET /api/v1/system/info - 获取系统信息
  • GET /api/v1/stats/summary - 获取系统统计

3. API设计原则

3.1 RESTful设计

  • 使用标准HTTP方法（GET、POST、PUT、DELETE）
  • 资源导向的URL设计
  • 状态码语义化

3.2 统一响应格式

所有API接口遵循统一的响应格式，便于前端统一处理。

3.3 异步处理

所有API接口支持异步处理，提高系统并发性能。

3.4 错误处理

统一的错误响应格式，包含错误码和详细信息，便于问题定位。
